## 주요 내용

- **진행 방식**: **Write for the reader** — Who is reading this and what do they need? **Start with the most useful information** — Don't bury the lede **Show, don't tell** — Code examples, commands, screenshots **Keep it current** — Outdated docs are worse than no docs **Link, don't duplicate** — Reference other docs instead of copying
