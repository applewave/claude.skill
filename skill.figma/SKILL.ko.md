# Figma MCP

Figma 기반 구현을 위해 Figma MCP 서버를 사용합니다. 설정 및 디버깅 세부 사항(환경 변수, 구성, 검증)은 `references/figma-mcp-config.md`를 참조하세요.

## Figma MCP 통합 규칙
이 규칙은 프로젝트에서 Figma 입력을 코드로 변환하는 방법을 정의하며, 모든 Figma 기반 변경 시 반드시 따라야 합니다.

### 필수 플로우 (건너뛰지 마세요)
1. 먼저 get_design_context를 실행하여 정확한 노드의 구조화된 표현을 가져옵니다.
2. 응답이 너무 크거나 잘린 경우, get_metadata를 실행하여 상위 수준의 노드 맵을 가져온 다음 get_design_context로 필요한 노드만 다시 가져옵니다.
3. 구현 중인 노드 변형의 시각적 참조를 위해 get_screenshot을 실행합니다.
4. get_design_context와 get_screenshot을 모두 확보한 후에만, 필요한 에셋을 다운로드하고 구현을 시작합니다.
5. 출력(일반적으로 React + Tailwind)을 프로젝트의 컨벤션, 스타일, 프레임워크에 맞게 변환합니다. 가능한 한 프로젝트의 색상 토큰, 컴포넌트, 타이포그래피를 재사용하세요.
6. 완료로 표시하기 전에 Figma와 1:1 외관 및 동작을 검증합니다.

### 구현 규칙
- Figma MCP 출력(React + Tailwind)은 디자인과 동작의 표현으로 취급하며, 최종 코드 스타일이 아닙니다.
- 해당되는 경우 Tailwind 유틸리티 클래스를 프로젝트의 선호 유틸리티/디자인 시스템 토큰으로 교체합니다.
- 기능을 중복하지 말고 기존 컴포넌트(버튼, 입력, 타이포그래피, 아이콘 래퍼 등)를 재사용합니다.
- 프로젝트의 색상 시스템, 타이포그래피 스케일, 스페이싱 토큰을 일관되게 사용합니다.
- 저장소에 이미 적용된 기존 라우팅, 상태 관리, 데이터 페치 패턴을 존중합니다.
- Figma 디자인과 1:1 시각적 일치를 추구합니다. 충돌이 발생하면 디자인 시스템 토큰을 우선하고 시각적 일치를 위해 스페이싱이나 크기를 최소한으로 조정합니다.
- 최종 UI를 Figma 스크린샷과 외관 및 동작 모두에 대해 검증합니다.

### 에셋 처리
- Figma MCP 서버는 이미지 및 SVG 에셋을 제공할 수 있는 에셋 엔드포인트를 제공합니다.
- 중요: Figma MCP 서버가 이미지나 SVG에 대해 localhost 소스를 반환하면, 해당 이미지 또는 SVG 소스를 직접 사용하세요.
- 중요: 새로운 아이콘 패키지를 가져오거나 추가하지 마세요. 모든 에셋은 Figma 페이로드에 있어야 합니다.
- 중요: localhost 소스가 제공된 경우 플레이스홀더를 사용하거나 생성하지 마세요.

### 링크 기반 프롬프팅
- 서버는 링크 기반입니다: Figma 프레임/레이어 링크를 복사하여 구현 도움을 요청할 때 MCP 클라이언트에 해당 URL을 제공하세요.
- 클라이언트는 URL을 탐색할 수 없지만 링크에서 노드 ID를 추출합니다. 항상 링크가 원하는 정확한 노드/변형을 가리키는지 확인하세요.
