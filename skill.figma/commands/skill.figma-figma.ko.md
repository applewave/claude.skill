## Figma MCP 통합 규칙

이 규칙은 프로젝트에서 Figma 입력을 코드로 변환하는 방법을 정의하며, 모든 Figma 기반 변경 시 반드시 따라야 합니다.

### 필수 플로우 (건너뛰지 마세요)

| 순서 | 단계 | 설명 |
|:----:|------|------|
| 1 | `get_design_context` | 정확한 노드의 구조화된 표현을 가져옵니다 |
| 2 | `get_metadata` (필요 시) | 응답이 크거나 잘린 경우 상위 수준 노드 맵을 가져온 후 필요한 노드만 다시 가져옵니다 |
| 3 | `get_screenshot` | 구현 중인 노드 변형의 시각적 참조를 가져옵니다 |
| 4 | 에셋 다운로드 | get_design_context + get_screenshot 확보 후 필요한 에셋을 다운로드하고 구현 시작 |
| 5 | 코드 변환 | 출력(React + Tailwind)을 프로젝트 컨벤션, 스타일, 프레임워크에 맞게 변환 |
| 6 | 검증 | Figma와 1:1 외관 및 동작을 검증 후 완료 처리 |

### 구현 규칙

| 규칙 | 설명 |
|------|------|
| MCP 출력 취급 | Figma MCP 출력(React + Tailwind)은 디자인/동작의 표현이며, 최종 코드 스타일이 아닙니다 |
| Tailwind 교체 | 프로젝트의 선호 유틸리티/디자인 시스템 토큰으로 교체합니다 |
| 컴포넌트 재사용 | 기능 중복 없이 기존 컴포넌트(버튼, 입력, 타이포그래피 등)를 재사용합니다 |
| 토큰 일관성 | 프로젝트의 색상 시스템, 타이포그래피 스케일, 스페이싱 토큰을 일관되게 사용합니다 |
| 기존 패턴 존중 | 기존 라우팅, 상태 관리, 데이터 페치 패턴을 존중합니다 |
| 시각적 일치 | Figma 디자인과 1:1 시각적 일치를 추구합니다 (충돌 시 디자인 시스템 토큰 우선) |
| 최종 검증 | 최종 UI를 Figma 스크린샷과 외관 및 동작 모두에 대해 검증합니다 |

### 에셋 처리

| 규칙 | 설명 |
|------|------|
| 에셋 엔드포인트 | Figma MCP 서버가 이미지 및 SVG 에셋을 제공합니다 |
| **localhost 소스** | MCP 서버가 localhost 소스를 반환하면 **직접 사용**하세요 |
| **아이콘 패키지 금지** | 새로운 아이콘 패키지를 가져오거나 추가하지 마세요. 모든 에셋은 Figma 페이로드에 있어야 합니다 |
| **플레이스홀더 금지** | localhost 소스가 제공된 경우 플레이스홀더를 사용하거나 생성하지 마세요 |

### 링크 기반 프롬프팅

| 항목 | 설명 |
|------|------|
| 링크 기반 서버 | Figma 프레임/레이어 링크를 복사하여 MCP 클라이언트에 URL을 제공하세요 |
| 노드 ID 추출 | 클라이언트는 URL을 탐색할 수 없지만 링크에서 노드 ID를 추출합니다 |
| 정확한 링크 | 항상 링크가 원하는 정확한 노드/변형을 가리키는지 확인하세요 |
